# Руководство для авторов (Contributing)

Добро пожаловать! Мы рады, что вы хотите внести вклад в проект учебных материалов.

## Стандарты оформления документации

Для поддержания качества и единообразия материалов, пожалуйста, следуйте приведенным ниже шаблонам и правилам.

### Правила именования файлов

- **Лабораторные работы:** `lab-XX-topic.md` или `labXX/index.md` (где XX - номер работы).
- **Курсовые работы:** `XX-topic.md` (где XX - порядковый номер в курсе, например `01-rp-op-design.md`).
- Используйте `kebab-case` для названий файлов и папок.

### Обязательные секции

Каждый файл должен начинаться со следующей структуры:

```markdown
# [Название работы]

## Для кого этот документ
- Студентам X семестра
- Требования: [знания, навыки]
- Необходимое ПО: [список]

## Цель работы
[Краткое описание цели]

## Что нужно знать перед началом
- [Ссылка на теорию]
- [Ссылка на предыдущую работу]

## Где найти файлы проекта
Все файлы находятся в папке: `[путь]`
```

В конце каждого файла должны быть:

```markdown
## Чеклист выполнения
- [ ] Пункт 1
- [ ] Пункт 2

## Часто задаваемые вопросы (FAQ)
**Q: Вопрос?**
A: Ответ.

## Связанные материалы
- [Ссылка 1]
- [Ссылка 2]
```

### Шаблон для Лабораторной работы

```markdown
# Лабораторная работа №X: [Название]

## Для кого этот документ
...

## Цель работы
...

## Теоретическая часть
(Или ссылка на отдельный файл с теорией)

## Подготовка к работе
1. Установите ПО...
2. Скачайте шаблон...

## Задание
1. Выполнить...
2. Реализовать...

## Порядок выполнения
1. Шаг 1
2. Шаг 2

## Контрольные вопросы
1. Вопрос 1?

## Чеклист выполнения
- [ ] ...

## FAQ
...
```

```markdown
- [ ] ...

...
```

## Как внести вклад

### Сообщить об ошибке
1. Создайте issue в [GitHub репозитории](https://github.com/efremovnv/docs)
2. Опишите проблему подробно
3. Укажите, где именно находится ошибка

### Предложить улучшение
1. Создайте issue с описанием вашего предложения
2. Обсудите его с мейнтейнерами
3. После одобрения создайте Pull Request

## Структура коммитов
Используйте понятные сообщения коммитов:
- `docs: исправлена ошибка в lab1/task.md`
- `feat: добавлена лабораторная работа 3`
- `style: обновлен формат заголовков`
