Лабораторная работа 5. Взаимодействие программных компонентов друг с другом. Использование стека, процедур, функций и вложенных подпрограмм

Цель работы:​

Изучить, каким образом выполняется взаимодействие программных компонентов друг с другом; уяснить, как работает стек и как в процессорной системе можно использовать исходные данные, размещенные в отдельном файле.

Объекты изучения:​

  • стек;

  • команды процессора call, callr, ret;

  • утилиты Memory fill и Load file into memory приложения IMP для работы с оперативной памятью;

  • компоновщик программ.

Планируемые результаты обучения:​

После выполнения этой работы студенты смогут:

  • Создавать приложения, состоящие из нескольких программных компонентов, размещенных в разных файлах;

  • Организовывать взаимодействие программных компонентов через регистры процессора и через стек;

  • Использовать вложенные процедуры и функции;

  • Использовать в своих приложениях данные, размещенные в отдельных файлах.

Файлы, используемые в работе​

В этой работе используется предварительно созданная процессорная система, с которой студенты имели дело в предыдущих лабораторных работах.

В качестве исходного файла предлагается использовать файл с программой нахождения наибольшего числа из списка, подготовленный в соответствии с индивидуальным вариантом задания при выполнении пункта 2.3 лабораторной работы номер два. Студенты, которые смогли выполнить пункт 2.8 той же работы могут использовать программу нахождения наибольшего и наименьшего чисел из списка.

Подготовка к лабораторной работе​

  1. Изучите по учебной литературе, как выполняется передача параметров процедурам и функциям, и как обратно возвращается результат. Поместите в отчет краткое описание.

  2. Ознакомьтесь с командами вызова процедур и функций call, callr, и возврата из них ret [12,3]. Выясните, к какому формату относятся эти команды, и как происходит их выполнение. Поместите в отчет их краткое описание, включая их формат.

  3. Изучите, что собой представляет стек, и какие команды используются для работы с ним. Какой из регистров процессора используется для работы со стеком в процессорной системе? Поместите в отчет эту информацию, включая форматы вышеуказанных команд.

  4. Изучите утилиты Memory fill и Load file into memory приложения IМР [10,11]. Выясните, как заполнить диапазон памяти процессорной системы некоторыми константами с помощью утилиты Memory fill и как загрузить файл с исходными данными в оперативную память процессорной системы с помощью второй утилиты. Поместите эти материалы в отчет.

  5. Уясните содержание пунктов задания, выполняемого в текущей лабораторной работе.

  6. Подготовьте файл с исходными данными, соответствующими индивидуальному варианту задания для выполнения пункта 4.1.

  7. Подготовьте программные заготовки для выполнения остальных пунктов задания (2.1, 3.1, 5.1, 6.1, 6.2).

  8. Уясните индивидуальное задание. Подготовьте программу для его выполнения.

Вопросы для самоконтроля​

  1. Что такое стек, для чего он нужен, и как он работает?

  2. Какой способ адресации используется для обращения к стеку?

  3. Какие команды и какого формата используются для работы со стеком?

  4. Как передаются параметры вызываемым процедурам и функциям? Как возвращается результат?

  5. Какие команды и какого формата используются для вызова процедур и функций, и для возврата управления обратно?

  6. Какой способ адресации используется в командах вызова и возврата?

  7. Что такое вложенная подпрограмма?

  8. Каким образом следует организовать корректный возврат из вложенных подпрограмм?

  9. Можно ли исходные данные для программы разместить в отдельном файле? Как это сделать?

  10. Как с помощью приложения IMP загрузить в оперативную память процессорной системы данные из файла, начиная с заданного адреса?

  11. Как заставить программу обрабатывать загруженные из файла исходные данные?

  12. Если переместить исполняемый код программы в другую область оперативной памяти, будет ли программа корректно работать?

  13. Какие команды перестанут корректно выполняться после перемещения программы в оперативной памяти?

  14. Можно ли программный код использовать как исходные данные в другой программе? Приведите такой пример.

  15. Как будет скомпонована программа, состоящая из отдельных компонентов, размещенных в разных файлах?

Порядок выполнения лабораторной работы​

Часть 1. Передача параметров через регистры процессора​

Используйте подготовленную вами во второй лабораторной работе в соответствии с индивидуальным вариантом задания программу нахождения наибольшего числа из списка, либо наибольшего и наименьшего числа, для тех студентов, которые с этим заданием справились.

1.1. Запустите приложение Intel Monitor Program. Откройте проект, созданный во второй лабораторной работе. Выполните программу нахождения наибольшего числа из списка. Убедитесь в работоспособности программы, меняя исходные числа во вкладке Memory IMP и наблюдая полученный результат.

1.2. Часть кода, в которой выполняется поиск максимального числа из списка оформите в виде подпрограммы с именем Max. Для передачи параметров подпрограмме используйте регистры r5, для указания числа слов в списке, и регистр r6, для указания начального адреса списка. Подпрограмма будет осуществлять анализ содержимого ячеек памяти, и находить наибольшее число из списка. Потом она должна поместить результат в регистр r7 и завершиться командой ret.

1.3. Список обрабатываемых чисел разместите в отдельной секции данных в начале статической памяти. В основной программе заполняются регистры r5 и r6 для передачи параметров, и осуществляется вызов подпрограммы Мах с использованием команды call. Затем, после возврата результата в r7 она выводит найденное число на красные светодиоды. Проверьте правильность работы программы, визуально наблюдая содержимое анализируемой ею области памяти, используя вкладку Memory, содержимое регистра r7 в регистровом окне приложения IMP и младшую часть результата, отображаемую на светодиодах. Рекомендуется поставить контрольные точки на командах call и ret и сами эти команды выполнить по шагам, осмысливая производимые ими действия.

1.4. Внесите изменения в основную программу, так чтобы вызов подпрограммы выполнялся с использованием команды callr.

1.5. Поместите в отчет машинные коды команд call, callr и ret, наблюдаемые во вкладке Disassembly. При подготовке отчета выполните их декодирование и осмыслите содержимое отдельных полей этих команд.

1.6. Оформите вывод результата на красные светодиоды в виде самостоятельной процедуры Out_on_LEDR, передав ей в качестве параметра отображаемое число через какой-либо регистр процессора. Например, через r7.

1.7. Включите в основную программу команду вызова этой процедуры. Наблюдайте результат работы программы. Поместите в отчет схему взаимодействия основной программы с подпрограммами. Основная программа начинается с метки _start и завершается командой stop и содержит две команды call. Первая вызывает функцию Max, которая размещается в ОП с метки Max. Вторая вызывает процедуру Out_on_LEDR, размещенную в ОП с одноименной метки. Обе подпрограммы завершаются командой ret. Первая возвращает найденный результат основной программе в регистре процессора. Вторая в основную программу никакой результат не возвращает, она выводит переданный ей параметр на светодиоды.

Часть 2. Использование вложенных подпрограмм​

2.1. Модифицируйте программу из предыдущей части следующим образом. Выполните вывод найденного числа на светодиоды вызвав процедуру Out_on_LEDR внутри подпрограммы Max.

2.2. Поставьте контрольную точку в конце основной программы. Запустите программу. Убедитесь, что возврат в основную программу не выполняется. Выясните причину и устраните ее. Для этого на командах call и ret рекомендуется поставить контрольные точки и выполнить их по шагам, осмысливая выполняемые ими действия.

2.3. Убедитесь в работоспособности исправленной программы. Покажите результат ее работы преподавателю. Поместите в отчет листинг отлаженной программы и схему взаимодействия основной программы с обеими подпрограммами.

Часть 3. Передача параметров через стек​

3.1. Модифицируйте основную программу из первой части таким образом, чтобы передача параметров функции Max осуществлялась через стек. Для этого в основной программе определите в качестве вершины стека адрес последнего слова используемой в процессорной системе динамической памяти SDRAM. В качестве указателя стека используйте регистр sp процессора. Обратите внимание на то, что после помещения в стек или извлечения из стека автоматического изменения регистра sp не происходит. Возврат найденного максимального числа из списка также должен осуществляться через стек. Причем результат должен быть помещен в то место в стеке, где перед вызовом были размещены передаваемые функции параметры. Для этого используйте соответствующее смещение относительно текущего указателя стека.

3.2. Общепринятый прием написания подпрограмм — сохранение содержимого регистров, используемых в подпрограмме, в стеке в начале работы подпрограммы и восстановление их перед завершением подпрограммы. Непосредственно перед выполнением команды ret выполните коррекцию указателя стека так, чтобы он указывал на найденный подпрограммой результат. Чтобы визуально наблюдать корректное восстановление используемых в подпрограмме регистров, рекомендуется в основной программе перед командой вызова поместить в них выразительные значения, например: в регистр r5 число 555, в регистр r6 число 666, в регистр r7 число 777 и т.д. Отладьте программу, убедитесь в ее работоспособности, покажите результат её работы преподавателю. Поместите в отчет листинг отлаженной программы и зарисовку стека перед выполнением команды call и после выполнения команды ret.

3.3. Оформите процедуру Out_on_LEDR с учетом рекомендаций из предыдущего пункта задания. То есть так, чтобы она не могла искажать содержимое используемых в ней регистров после своего завершения.

3.4. Запускайте программу, изменяя исходные данные. Чтобы повторно не компилировать программу, изменяйте числа в списке непосредственно в ОП, используя вкладку Memory приложения IMP. Следует напомнить, что список чисел размещен в начале статической памяти.

Часть 4. Использование файлов с исходными данными в IMP​

4.1. Создайте файл и поместите в него подготовленный в работе 2, в соответствии с индивидуальным вариантом задания, список чисел. Для этого используйте любой текстовый редактор и шестнадцатеричную форму представление чисел. В качестве разделителя может использоваться любой символ. Загрузите файл в оперативную память, начиная с заданного адреса. Используйте команду Load file into memory из меню Actions приложения IMP. В появившемся окне задайте имя файла с указанием пути к нему, в поле «формат файла» окна браузера задайте Delimited hexadecimal value file, символ разделителя и начальный адрес загрузки в оперативную память.

4.2. Рассмотрите возможность загрузки файла как отдельно от кода программы, например, в статическую память SRAM, так и в непосредственной близости к программному коду, в SDRAM. Убедитесь в правильности загрузки списка чисел в ОП, используя вкладку Memory.

4.3. Модифицируйте программу из третьей части таким образом, чтобы она передавала подпрограмме Max число слов в списке, соответствующее подготовленному вами файлу и начальный адрес ОП, куда был загружен список чисел. После возврата результата в основную программу, последняя выводит его на светодиоды, используя процедуру Out_on_LEDR. Покажите результат работы программы преподавателю.

Часть 5. Компоновка программы из нескольких файлов​

5.1. Поместите исходный код подпрограмм Max и Out_on_LEDR в отдельные файлы с одноименным названием. Объявите метки Max и Out_on_LEDR, с которых начинается выполнение подпрограмм, как глобальные имена. Используя вкладку Program Settings приложения IMP, добавьте вновь созданные файлы к основной программе. Выполните компиляцию и загрузку программы в ОП процессорной системы. С помощью вкладок Disassembly и Memory найдите в ОП секции кода основной программы и двух её подпрограмм, и секцию данных со списком чисел. Отразите в отчете их место расположения в памяти. Для удобства поиска разместите секцию данных в статической памяти SRAM. Запустите программу и убедитесь в успешном её выполнении, наблюдая найденный программой результат на красных светодиодах.

5.2. Добавьте в каждую из указанных подпрограмм секции данных. Поместите в них какие-нибудь значения, например, текстовые строки с названием подпрограмм. Выполните компиляцию и загрузку программы в ОП процессорной системы. С помощью вкладок Disassembly и Memory найдите в ОП секцию кода основной программы и двух её подпрограмм, и секцию данных со списком чисел и с двумя текстовыми строками. Сделайте снимок экрана и поместите его в отчет, снабдив вашими комментариями о взаимном размещении в ОП. Выполните программу, наблюдайте найденный ею результат на красных светодиодах.

5.3. Измените порядок перечисления программных компонентов во вкладке Program Settings. Первой укажите процедуру Out_on_LEDR, затем функцию Max, а затем основную программу. Экспериментально определите к каким последствиям это приведет. Используя проводник найдите в папке с проектом исполняемый файл скомпонованной программы. Зафиксируйте в отчете его название и начальный адрес, с которого начнется выполнение программы. Запустите программу и убедитесь в корректном её завершении.

Часть 6. Перемещение кода программы в ОП, и оценка корректности его выполнения​

6.1. Напишите процедуру Copy, которая выполняет копирование одной области ОП в другую область памяти, начиная с заданного адреса. Используйте для неё следующие параметры: начальный и конечный адреса копируемого фрагмента оперативной памяти и адрес в ОП, с которого следует разместить создаваемую копию. Поместите исходный код этой процедуры в новый файл и сохраните его с новым именем, отражающим его содержимое. Например, «Copy»

6.2. Напишите программу Main, которая с помощью процедуры Copy будет копировать секцию кода, включающую основную программу нахождения максимального числа из списка и две подпрограммы Max и Out_on_Ledr в ОП со смещением 0х1000 от начала секции кода. Объявите метку Main, с которой начинается ваша программа, глобальной. Поместите исходный код этой программы в отдельный файл с таким же именем. В Memory Settings укажите метку Main в качестве стартового адреса новой программы.

6.3. Во вкладке Program Settings добавьте вновь созданные файлы к файлам, использованным ранее для выполнения пункта 5.2 задания. Выполните компиляцию и загрузку обновленной программы, включающей теперь уже пять компонентов из пяти файлов. Используя вкладки Disassembly и Memory найдите в ОП секцию кода, состоящую из пяти компонентов и секцию данных со списком чисел и текстовыми строками. Отразите в отчете их взаимное расположение в ОП.

6.4. Убедитесь, что текущий адрес в РС указывает на начало вновь созданной программы, то есть на метку Main. Следует напомнить, что во вкладке Disassembly текущая команда выделяется желтым цветом. Поместите её адрес в отчет.

6.5. Поставьте контрольную точку в конце программы Main и выполните её в автоматическом режиме.

6.6. Используя вкладку Memory визуально убедитесь, что копирование программы нахождения максимального числа из списка вместе с двумя подпрограммами Max и Out_on_Ledr выполнилось правильно.

6.7. Попробуйте выполнить скопированный код этой программы, предварительно установив в счетчике команд его адрес. Проверьте правильность перехода к вызываемым функции и процедуре и возврата из них. Для этого рекомендуется поставить контрольные точки на командах call, ret и на последней команде основной программы с меткой stop. Включите в отчет ваши наблюдения и объяснения.

6.8*. Добавьте к программным компонентам Copy и Main секции данных, определив в них некоторые числовые значения и(или) текстовые строки, например, с названием соответствующей программы. Следует напомнить, что программа нахождения наибольшего числа из списка содержит в секции данных сам список чисел. Скомпилируйте программу и загрузите её в ОП. Используя вкладку Memory найдите в ОП программный код и данные, определенные в разных файлах. Отразите в отчете ваше наблюдение.

Отчетные материалы​

Отчетные материалы должны содержать:

  1. Цель лабораторной работы.

  2. Материалы, связанные с подготовкой к работе, включая теоретическую часть.

  3. Информацию по выполнению каждого пункта задания. Причем в отчете должны содержаться выполняемые вами действия, наблюдаемые результаты, и ваши объяснения.

  4. Написанные вами фрагменты программ, включая программу для выполнения индивидуального задания из приложения В.

  5. Зарисовку ОП с отображением в ней пяти программных компонентов, обрабатываемого списка чисел, функции Max, процедур Out_on_LEDR, Copy и стека.

  6. Краткое заключение.

Примеры некоторых вопросов и заданий для защиты лабораторной работы​

Для защиты работы студенты должны продемонстрировать умение:

  • Создавать приложения, состоящие из нескольких программных компонентов, размещенных в разных файлах;

  • Управлять компоновкой объектных файлов с целью создания исполняемого кода программы;

  • Организовывать взаимодействие программных компонентов друг с другом как через регистры процессора, так и через стек;

  • Использовать вложенные подпрограммы;

  • Использовать в своих приложениях исходные данные, размещенные в отдельных файлах;

  • Выполнять перемещение программного кода в ОП и оценивать возможность его корректного выполнения.

    Ниже в приложении В приводятся задания, которые могут быть предложены студентам для самостоятельного выполнения. Успешное их выполнение является достаточным условием для защиты лабораторной работы.

Приложения​

Приложение В. Задания для самостоятельного выполнения.​

  1. Напишите программу, которая использует рекурсию для вычисления факториала числа N. N!=N(N-1)(N-2)…*3*2*1. Заметьте, что N!=N*(N-1)! Программа включает подпрограмму Factor, которая вызывает сама себя до тех пор, пока желаемый факториал не будет получен. Основная программа должна ввести параметр N, задаваемый с помощью переключателей, затем передать N как параметр, размещая его в стеке. Подпрограмма может вернуть результат через регистр r4, либо через стек. Программа Factor вызывает сама себя. Перед вызовом не забудьте сохранить адрес возврата (регистр ra) в стеке и восстановить его после возврата. Покажите результат работы программы преподавателю.

  2. Напишите программу, которая осуществляет вывод на LCD содержимого области памяти, представленного в шестнадцатеричном виде. Оформите ее в виде подпрограммы, которой передаются параметры: начальный адрес области памяти, количество отображаемых байтов и номер начальной позиции на LCD дисплее.

  3. Напишите программу, которая отображает содержимое 32-разрядного регистра процессора на 7-сегментных индикаторах HEX7-HEX0 в виде шестнадцатеричного числа. Оформите ее в виде процедуры HEX_Display, которой в качестве параметра передается выводимое значение через регистр процессора.

  4. Напишите программу, которая осуществляет вывод на 7-сегментные индикаторы содержимого области памяти, представленного в шестнадцатеричном виде. Оформите ее в виде подпрограммы, которой передаются параметры: начальный адрес области памяти, количество отображаемых байт (не более 4). Младшие байты отображаются справа.

  5. Напишите программу, которая осуществляет упорядочивание чисел в списке в убывающем порядке. Каждое число представляет собой 32-разрядное число без знака. Оформите ее в виде подпрограммы, которой передаются параметры: адрес списка в ОП, число слов в списке.

  6. Напишите программу, вычисляющую первые n чисел Фибоначчи. Числа 32 – разрядные. Они вычисляются по формуле An=An-1+An-2 и записываются в оперативную память. Первые числа выглядят следующим образом: 0,1,1,2,3,5,8…. Оформите в виде подпрограммы, которой передаются параметры: адрес ОП, число n.

  7. Напишите программу, вычисляющую максимальное количество повторяющихся нулевых и единичных разрядов в числе, хранящемся в ОП. Адрес числа может быть произвольным. Оформите в виде подпрограммы, передав в качестве параметров: адрес ОП и размер числа в байтах (не более 4). Вывод найденных значений выполните на красные и зеленые светодиоды.

  8. Напишите программу, вычисляющую количество положительных и отрицательных фронтов в числе, хранящемся в ОП. Например, число 0b10010110 содержит 2 положительных и 3 отрицательных фронта. Адрес числа может быть произвольным. Оформите в виде подпрограммы, передав в качестве параметров адрес ОП и размер числа в байтах (не более 4). Вывод найденных значений выполните на красные и зеленые светодиоды.

  9. Оформите в виде отдельной процедуры программу, выполняющую вывод на LCD экран текстовой информации. В качестве параметров используйте начальный адрес в ОП выводимой строки и координаты у и х, задающие начальную позицию на LCD. Параметры передавайте через стек.

  10. Напишите программу, которая выводит текстовую строку на LCD дисплей. Рекомендуется оформить её в виде процедуры, которой в качестве параметров передаются адрес в ОП выводимой текстовой строки и координаты y и x, начальной позиции на LCD, через регистры процессора.

  11. Напишите программу, которая выводит текстовую строку на LCD в режиме бегущей строки. Рекомендуется оформить её в виде процедуры, которой в качестве параметров передаются адрес в ОП выводимой текстовой строки и координаты y и x, начальной позиции на LCD, через регистры процессора. Эта процедура будет востребована в следующих лабораторных работах.
